# Product Context

## Пользовательская ценность

- **Прозрачность:** решение агента не только «в логах», но с on-chain якорем (hash, ссылка на off-chain доказательства — по дизайну).
- **Согласованность:** единый контракт того, что считается «интентом» и «решением».

## User journeys (черновик)

1. Внешняя система отправляет запрос с текстом/контекстом.
2. Сервис вызывает Gemini, получает структурированный результат (решение, обоснование, числовой `risk_level` 0–100).
3. После правил/лимитов формируется запись и отправляется транзакция в программу Anchor.
4. Клиент получает подтверждение и идентификаторы для аудита.
5. (День 3) Разработчик или агент через MCP вызывает те же возможности без ручного REST.

## Термины

- **Интент:** интерпретированное намерение (классификация, сущности, уверенность).
- **Решение:** то, что фиксируется on-chain (минимально: ссылка на интент + метаданные + время + authority).

## Нефункциональные ожидания

- Надёжность при сбоях RPC (retry, backoff).
- Безопасность ключей подписанта.
- Наблюдаемость (корреляционные id по цепочке Node ↔ Python ↔ Solana).
