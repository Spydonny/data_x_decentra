# KYA — Know Your Agent

AI-агенты уже управляют деньгами: оплачивают API, торгуют на DEX, управляют treasury. При этом отсутствует базовый слой доверия — невозможно определить, кто стоит за агентом и можно ли ему доверять.

**KYA (Know Your Agent)** — это on-chain слой верификации AI-агентов в экосистеме Solana.

## Идея

KYA переносит концепцию KYC на AI-агентов.

Перед выполнением любой операции агент:

* формирует intent (намерение действия)
* проходит проверку
* получает зафиксированное решение

Все решения записываются on-chain и формируют проверяемую репутацию агента.

## Как это работает

1. Агент формирует intent (адрес, сумма, действие)
2. Intent отправляется на проверку
3. LLM анализирует:

   * соответствие политике владельца
   * параметры транзакции
4. Выносится решение:

   * `APPROVE`
   * `REJECT`
   * `ESCALATE`
5. Решение записывается on-chain (PDA)
6. Транзакция исполняется или блокируется

Время обработки: ~2–3 секунды

## Архитектура

**Agent**

* инициирует действия
* накапливает репутацию

**KYA Service**

* валидирует intent
* принимает решение через LLM

**Solana Program**

* фиксирует решения on-chain
* хранит историю (PDA)
* обеспечивает прозрачность и проверяемость

**Observer / Regulator**

* читает историю
* анализирует поведение агента

## Почему Solana

* быстрая финальность
* низкая стоимость транзакций
* PDA для детерминированного хранения истории

## Зачем это нужно

AI-агенты уже выполняют финансовые операции, но инфраструктура доверия отсутствует.
KYA закрывает этот пробел, создавая стандарт верификации и репутации для AI-агентов.
